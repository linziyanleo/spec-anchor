# workflow_stop_dev - 停止项目

## 触发词

"停止项目" / "停止开发服务器" / "stop" / "关闭项目"

## 执行流程

1. 检查当前运行中的终端，识别开发服务器进程（npm run dev / vite / next dev 等）
2. 如有多个开发服务器，询问用户选择停止哪个或全部停止
3. 向对应终端发送终止信号（Ctrl+C）
4. 等待进程停止（最多 5 秒），超时则尝试 `kill -TERM <PID>`
5. 确认端口释放，告知用户停止结果

## 边界情况

- 无运行中的服务器：告知用户未检测到开发服务器
- 进程无法停止：尝试 `kill -KILL` 强制终止，仍失败则提供手动终止命令
- 后台进程：通过 `lsof -i :<port>` 检测常用端口（3000/5173/8080/4200）上的进程

## 参数

- `--force`：强制终止所有相关进程
- `--port`：指定要停止的端口号
- `--all`：停止所有检测到的开发服务器

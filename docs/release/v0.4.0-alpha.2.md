# v0.4.0-alpha.2 Release Note

## Summary

`v0.4.0-alpha.2` is the usage-proof prerelease for SpecAnchor.

It keeps the trustable core from `v0.4.0-alpha.1`, then adds runnable examples, smoke coverage, and walkthrough docs that prove an external user can actually install and use the public skill.

## What Changed

- Added committed examples for:
  - clean full-mode initialization
  - parasitic governance over external specs
  - agent prompts for Cursor, Claude Code, and Codex
- Added `tests/test_usage_proof.sh` and wired it into the public shell regression runner
- Added a dedicated CI job for usage-proof examples on Ubuntu and macOS
- Added `docs/USAGE_PROOF.md` and linked it from the public docs surface
- Added starter Global Specs during full-mode init so clean projects can pass strict health checks immediately
- Added `--format=summary` support to `specanchor-validate.sh`

## Installation Verification

From the repository root:

```bash
SPECANCHOR_SKILL_DIR="$PWD" bash scripts/specanchor-boot.sh --format=summary
bash scripts/specanchor-doctor.sh --strict
bash scripts/specanchor-validate.sh --format=summary
bash tests/run.sh
bash tests/test_usage_proof.sh
git diff --check
```

## For Agent Users

- Boot before editing files.
- Resolve anchors for the exact files and intent you plan to change.
- Report which Global / Module / Source anchors were used.
- If no anchor exists, stop and surface the gap instead of inventing product behavior.

## Known Limitations

- Examples are intentionally dependency-free and minimal.
- `resolve` is deterministic-first, not semantic search.
- This is still an alpha prerelease; public interfaces may change before `v0.4.0`.

## Upgrade Notes

- `v0.4.0-alpha.2` depends on the `v0.4.0-alpha.1` GitHub release/tag already being published first.
- Consumer install paths stay the same. Existing installs can be refreshed with the same `rsync --exclude-from=.skillexclude` flow.
- `specanchor-init.sh` now seeds starter Global Specs in full mode so strict doctor / validate can pass on a clean project.

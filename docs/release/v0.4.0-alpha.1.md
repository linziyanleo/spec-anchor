# v0.4.0-alpha.1 Release Note

## Summary

`v0.4.0-alpha.1` is the first public prerelease cut of SpecAnchor's trustable core.

It focuses on one thing: a public repository state that can be cloned, installed into another project, and sanity-checked with shell-based regression coverage.

## Highlights

- Self-bootable full-mode repo baseline
- Public shell regression runner (`bash tests/run.sh`)
- Fresh-clone smoke and consumer-install smoke in CI
- Natural-language-first command semantics, with `SA:` as optional shorthand
- `anchor.local.yaml` overlay support for maintainer-local sources
- Public docs and WHY entrypoints aligned to the tracked file layout

## Installation Verification

From the repository root:

```bash
SPECANCHOR_SKILL_DIR="$PWD" bash scripts/specanchor-boot.sh --format=summary
bash scripts/specanchor-doctor.sh --strict
bash tests/run.sh
git diff --check
```

## Known Limitations

- This is still an alpha prerelease. Public interfaces may change before the stable `v0.4.0` cut.
- `anchor.local.yaml` merge semantics are intentionally narrow: `sources` append, and public-script scalar reads prefer local values. It is not a generic deep-merge system.
- GitHub About metadata and GitHub Release publication require syncing this repo state to the public GitHub mirror.

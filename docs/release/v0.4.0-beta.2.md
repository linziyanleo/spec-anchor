# v0.4.0-beta.2 Release Note

## Summary

`v0.4.0-beta.2` migrates SpecAnchor from module-only indexing to the v3 `spec-index.md` contract, and moves SDD phase state out of task frontmatter.

## What Changed

- `specanchor-index.sh` now writes `.specanchor/spec-index.md` by default.
- `--legacy-module-index` keeps `.specanchor/module-index.md` available during migration.
- `frontmatter-inject.sh` can migrate `sdd_phase` into the body marker and normalize legacy task `status: active`.
- `specanchor-boot.sh` now prints compact `Available Commands` and `Available Modules` sections.

## Migration

```bash
bash scripts/frontmatter-inject.sh --migrate-sdd-phase --dir .specanchor/tasks
bash scripts/frontmatter-inject.sh --normalize-task-status --dir .specanchor/tasks
bash scripts/specanchor-index.sh --legacy-module-index
```

## Validation

```bash
bash scripts/specanchor-boot.sh --format=summary
bash scripts/specanchor-doctor.sh --strict
bash scripts/specanchor-validate.sh --strict
bash tests/run.sh
SPECANCHOR_RUN_BATS=1 bash tests/run_all.sh
```

## Known Limitations

- `module-index.md` remains as a deprecated fallback until the v0.5 removal window.
- Parasitic mode keeps its existing source mapping behavior.
